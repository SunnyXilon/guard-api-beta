# Acceptable Use Policy Draft

This draft is a starting point and should be reviewed before public launch.

Customers may not use Guard API to:

- Process content for unlawful harassment, surveillance, or discrimination.
- Attempt to identify, profile, or target protected classes unlawfully.
- Upload malware, exploit payloads, or credential dumps except in a lawful defensive security context.
- Circumvent authentication, billing, credit quotas, rate limits, or usage controls.
- Resell the service without a written agreement.
- Submit content that they do not have rights or authorization to process.

Guard API may suspend or limit accounts that create security, legal, platform, or operational risk.
